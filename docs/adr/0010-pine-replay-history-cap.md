# ADR-0010: Pine Replay History Cap at ADR Lookback Depth

The GxT Correlated Asset Indicator replays Pine runtime history on every chart load. TradingView profiling showed the 1m `request.security` session-rails path as the dominant hotspot when history is unbounded. Failure Swing comparison in the domain glossary scans four CME weeks of HTF swings — wider than the two-week display lookback — so outer extremes can stamp nearby new swings even after they age out of the canvas.

We cap replay at **14 completed CME daily sessions** (`ADR_LOOKBACK`). Runtime scan depth is timeframe-aware via `session_bar_cap = ADR_LOOKBACK × 24h / chart bar duration`; loops use `BAR_TIME_SEARCH_MAX = min(4999, session_bar_cap − 1)`. Pine CE10123 requires **const int** for `indicator(calc_bars_count)` and `max_bars_back`, so those declarations use fixed literals: `calc_bars_count = 20160` (14 sessions at 1m, the binding session-rails TF) and `max_bars_back(time, 5000)` (Pine CE ceiling + 1). That depth matches ADR warm-up and keeps session rails / Session Context math honest; Active DOL targets are typically PD rails or same-week liquidity, so the shorter window is sufficient for live trading geometry. When a level origin timestamp falls before searchable history, line starts clip to the oldest loaded bar instead of defaulting to the active bar.

**Consequence:** replay is faster and session rails draw at correct prices on capped history, but Failure Swing suppression is best-effort for swings whose outer comparison extremes formed outside the 14-session replay window — those failure swings may still print.
